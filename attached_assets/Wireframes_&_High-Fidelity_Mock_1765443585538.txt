* **System Name:** MDRRMO Pio Duran Maps
* **Layout:** The interface must dedicate maximum screen space to the map itself, treating it as the core element. The map control/layer menu (`Interactive Map`, `Topographic Map`, etc.) should be presented as a clean, dismissible, or collapsible sidebar.

* **Layer Organization:** The menu items (`Interactive Map`, `Administrative Map`, etc.) must function as toggle buttons, not just simple links.

# ‚òéÔ∏è Contact Directory Design Specification

## I. Global Screen Aesthetic and Organization
fetch on google sheet: https://docs.google.com/spreadsheets/d/11uutE9iZ2BjddbFkeX9cQVFOouphdvyP000vh1lGOo4/edit?usp=sharing
Sheet3 column:  Status Indicator, Contact Name, Role/Title, Agency, Primary Phone


* **Background:** Maintain the consistent deep Navy/Indigo background (`#1A1E32`).
* **Goal:** The design must allow users to locate a contact, agency, or specific emergency number in under three seconds. The layout is optimized for rapid scanning.
* **Structure:** Primarily composed of four key sections:
    1.  **Summary Statistics** (Top Banner).
    2.  **Search & Filter Tools**.
    3.  **Directory View** (The main list/grid of contacts).

## II. Summary Statistics & Contact Cards (`contact information.jpg` Enhancement)

The four summary metric cards must be prominent and visually reinforcing.

* **Card Design:** Use large, clear, custom icons and a font size that makes the zero-state (or current count) instantly readable.
* **Interaction:** These cards can also function as quick filters (e.g., clicking the "Agencies" card filters the main directory to show only agencies).

| Metric | Card Color/Icon Theme |
| :--- | :--- |
| **Total Contacts** | Primary Accent Color (Teal) / User Icon |
| **Agencies** | Secondary Accent Color (Lime Green) / Building Icon |
| **Phone Numbers** | Violet/Purple / Phone Icon |
| **Email Addresses** | Amber/Orange / Envelope Icon |

## III. Search, Filtering, and Sorting

The tools needed for quick contact retrieval must be intuitive and powerful.

* **Intelligent Search:** The search bar (`Search contacts by name, agency, or role...`) should be prominent and support partial matching and auto-suggestions.
* **Critical Filters:** Implement quick-access filter buttons (or a stylized dropdown) positioned near the search bar:
    * **Filter by Role:** (e.g., *MDRRMO Head, First Responder, Logistics*).
    * **Filter by Priority Level:** (e.g., *Level 1: Critical Emergency, Level 2: Support*).
    * **Filter by Status:** (e.g., *Active, On Leave*).
* **Sorting:** Default sort should be by **Name (A-Z)**, with options for sorting by **Agency** or **Last Updated**.

## IV. Directory View (The Main List)

The contacts must be presented in a clean, high-density list view for efficient scanning.

* **Row Styling:** Each contact should occupy a clean, horizontally-oriented "Contact Row Card" with minimal separators, focusing on white space for readability.
* **Key Information Display (Left-to-Right):**
    1.  **Status Indicator:** A small, circular colored badge (Green for available, Red for emergency contact, Yellow for advisory role) placed next to the contact name.
    2.  **Contact Name:** Clear and bolded.
    3.  **Role/Title:** Subdued, smaller text under the name.
    4.  **Agency:** Prominent display of the associated agency (e.g., *PNP, BFP, LGU Health*).
    5.  **Primary Phone:** The most critical number, displayed clearly.
    6.  **Action Icons:** Small, non-text icons for **`Call`** and **`Email`** on the far right.

### Emergency Contact Feature (Highlighting)

* **Design:** Contacts marked as **Critical Emergency** must be instantly recognizable. This can be achieved by giving the entire row a subtle red border or a distinct, slightly darker background tint.
* **Functionality:** A dedicated filter/toggle to show only **Emergency Contacts** should be the first option available.
